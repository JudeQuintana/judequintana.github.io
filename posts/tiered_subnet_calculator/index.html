Tiered Subnet Calculator in Terraform

Dec 1, 2020
6 min read

I’ve been thinking about adding support for building tiered subnets of any size for the next iteration of the Dynamic VPC module. It occurred to me that keeping the auto subnet generation inside the module actually makes the subnetting less dynamic.

Furthermore, auto subnet calculation should be in assistance to the process of allocating subnets and should not be fed directly as input to the VPC module. This is due to the fact that order matters only for the subnet calculation, so when tiers or azs are changed, added or removed, the calculation will shift for some or all tiers.

Removing the subnet calculation will simplify the module itself by doing less work, although, it probably should do some kind of subnet validation against a network cidr. This will also re-enforce the notion that engineers should know their subnetting when architecting networks. With all that in in mind, I made Tiered Subnet Calculator module to help with allocating subnets per AZ per network tier.

tiers.auto.tfvars

vpc_cidr_block = "10.0.0.0/16"

tiers = [
  {
    name   = "app"
    acl    = "public"
    newbit = 4
  },
  {
    name   = "db"
    acl    = "private"
    newbit = 4
  },
  {
    name   = "worker"
    acl    = "private"
    newbit = 4
  },
  {
    name   = "lbs"
    acl    = "public"
    newbit = 4
  }
]

az_newbits = {
  a = 4
  b = 4
  c = 4
  d = 4
}

variables.tf

variable "vpc_cidr_block" {
  type = string
}

variable "tiers" {
  type = list(object({
    name   = string
    acl    = string
    newbit = number
  }))
}

variable "az_newbits" {
  type = map(number)
}

main.tf

locals {
  # generate top level networks for each tier based on tier newbit + vpc_cidr_block mask ie /4 + /16 = /20
  tier_networks = zipmap(var.tiers[*].name, cidrsubnets(var.vpc_cidr_block, var.tiers[*].newbit...))

  # generate a subnet based on each az newbit (in azs_new_bits map) per tier network ie /4 + /20 = /24
  tier_subnets = { for t, n in local.tier_networks : t => cidrsubnets(n, values(var.az_newbits)...) }

  # generate azs to subnet map per tier
  tier_az_subnets = { for t, s in local.tier_subnets : t => zipmap(keys(var.az_newbits), s) }

  # build new tiers list with their associated az to subnets map
  tiers_with_subnets_per_az = [
    for t in var.tiers : {
      name    = t.name,
      acl     = t.acl,
      network = lookup(local.tier_networks, t.name)
      azs     = lookup(local.tier_az_subnets, t.name),
  }]

}

output "calculated_tiers" {
  value = local.tiers_with_subnets_per_az
}
